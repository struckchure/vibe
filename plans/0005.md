| Field | Value |

## Summary

1:1 voice and video calls from the chat thread header, reusing the existing encrypted gossipsub signaling path and per-contact `RTCPeerConnection` (same PC as `vibe/text`).

## Delivered

### UI

- [`chat-thread.tsx`](../src/components/chat/chat-thread.tsx): Phone and Video buttons in header; transport badge **Direct** / **Network**
- [`call-overlay.tsx`](../src/components/call/call-overlay.tsx): In-call UI (voice avatar / video PiP)
- [`incoming-call-dialog.tsx`](../src/components/call/incoming-call-dialog.tsx): Accept / Decline
- [`chat-layout.tsx`](../src/components/chat/chat-layout.tsx): Mounts call overlay and incoming dialog

### Media / signaling

- [`calls.ts`](../src/lib/calls.ts): `call-invite`, `call-answer`, `call-decline`, `call-end` over encrypted signaling
- [`webrtc.ts`](../src/lib/webrtc.ts): Shared peer connection, `ontrack` for remote media, call signal routing
- [`use-call.ts`](../src/hooks/use-call.ts): Hook for chat layout

### Platform

- [`tauri.conf.json`](../src-tauri/tauri.conf.json): iOS mic/camera usage strings; Android camera + record audio permissions

## Test plan

1. Two clients, same room, add contacts, open chat.
2. Voice call: ring → accept → audio both ways → end; text still works.
3. Video call: camera on/off, end call.
4. Decline incoming call.
5. Call while Direct vs Network transport.
6. iOS/Android permission prompts.
