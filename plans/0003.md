## Goal

Deliver 1:1 text over the `vibe/text` WebRTC data channel when connected, with encrypted signaling and gossipsub fallback.

## Transport

| Path | When |
|------|------|
| **WebRTC DC** | `RTCDataChannel` label `vibe/text` is `open` |
| **Gossipsub** | Fallback via `vibe/msg/<conversation_id>` when DC not ready |

Wire format on both paths: JSON `WireChat` (ChaCha20-Poly1305 ciphertext inside).

## Signaling

- Topic: `vibe/signal/<conversation_id>` (unchanged)
- SDP/ICE payloads encrypted with session key (`encrypt_signaling` / `decrypt_signaling`)
- Polite/impolite: higher `peerId` (base64url) is offerer; lower peer answers

## Rust

- [crypto.rs](../src-tauri/src/crypto.rs): `WireChat`, `build_wire_chat`, `ingest_wire_chat`, `encrypt_for_peer`
- Commands: `prepare_wire_message`, `persist_outgoing_message`, `encrypt_signaling`, `decrypt_signaling`, `ingest_dc_message`

## Frontend

- [webrtc.ts](../src/lib/webrtc.ts): connection manager, hybrid `sendTextMessage`
- Thread badge: **Connected** (DC) / **Relay** (gossipsub)

## Acceptance criteria

- [x] Polite/impolite negotiation avoids offer glare
- [x] Signaling encrypted on the wire
- [x] Text over DC when channel open
- [x] Gossipsub fallback when DC not open
- [x] Incoming DC messages in thread + contact preview
- [x] `cargo build` + `bun run build` pass

## Known limits

- Empty `iceServers`: best on same LAN; cross-NAT needs future STUN/relay
- Overlay must deliver gossipsub signaling before WebRTC can connect
- Full Noise `/vibe/noise/1` stream and CBOR envelopes deferred

## Manual test

1. Two clients, each with the other as a contact (same network or room for gossipsub)
2. Open chat; wait for **Connected** badge
3. Send messages; verify delivery
4. If DC never opens, messages still send with **Relay** badge
