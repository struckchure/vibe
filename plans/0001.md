# 0001 — Text, identity, and room discovery

| Field | Value |
|-------|-------|
| **Status** | done |
| **SPEC** | M0 (shell + 1:1 text) |

## Goal

First encrypted 1:1 text between two peers on the overlay, with a WhatsApp-style chat UI.

## Prerequisite (done)

- Tab shell at `/`, `/voice`, `/video` via TanStack Router
- shadcn + Tailwind + Lucide

## Scope

### UI

- Resizable two-pane chat (`ResizablePanelGroup`)
- `Empty` when no conversation selected
- Sidebar: conversation list (`Item` + `Avatar`)
- Thread: messages + `InputGroup` composer
- Dialogs: My peer ID, add contact, join room, room peer list

### Backend (Rust)

- Ed25519 identity key file on device (stable peer ID across restarts)
- In-memory contacts, messages, and session keys (cleared on app exit)
- libp2p gossipsub: room announce + signaling
- Noise XX session keys; WebRTC data channel `vibe/text` for ciphertext

### Out of scope

- IPFS profiles, groups, voice/video tabs, push, global directory

## Discovery

- **Peer ID** = Ed25519 public key (base64url)
- **Room code** → topic `vibe/room/<sha256(code||salt)>` + signed announce
- **Option A:** Room is lobby; messaging requires add contact

## Acceptance

- [x] Resize chat panes; empty state when no selection
- [x] Same room code → peers visible in room dialog
- [x] Add contact → appears in chat list
- [x] Two online clients exchange text via encrypted gossipsub (app foreground); WebRTC signaling scaffold for DC upgrade
