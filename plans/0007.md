# 0007 — Auto-connect via gossipsub (connected peers only)

| Field | Value |
|-------|-------|
| **Status** | done |

## Summary

Restore a minimal libp2p gossipsub overlay (no LAN/mDNS, no bootstrap/relay/DHT). When a contact is a **connected libp2p peer**, opening their chat auto-exchanges WebRTC SDP over `vibe/signal/<conversation_id>`. Manual `vibe://connect` links remain as an advanced fallback.

## Changes

- Slim [`network.rs`](../src-tauri/src/network.rs): gossipsub + identify + inbound TCP
- Overlay commands: `start_network`, `subscribe_conversation`, `publish_signaling`, `is_overlay_peer_connected`, `dial_contact`
- Asymmetric `derive_session_key` for pre-Noise signaling encryption
- [`ensureTextTransport`](../src/lib/transport/peer-connection.ts) + [`useAutoConnect`](../src/hooks/use-auto-connect.ts)
- Connect dialog moved to chat overflow menu

## Manual test

1. Establish libp2p TCP connection between two contacts (`dial_contact` with explicit multiaddrs, or inbound dial)
2. Open chat on both — auto WebRTC without connect links
3. Open chat before overlay peer connects — "Waiting for … on the libp2p overlay"
4. Fallback: Advanced → manual connect link

## Known limits

- Gossipsub signaling requires the contact in `connected_peers()`
- No LAN discovery or public bootstrap — peers need explicit multiaddr exchange (follow-up)
